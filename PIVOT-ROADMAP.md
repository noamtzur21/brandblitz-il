# BrandBlitz IL – פיבוט למוצר שעסק ישלם עליו

**מטרה:** להפוך מ"גימיק נחמד" (תוכן גנרי מ־AI) למוצר שבעל עסק ישראלי ישים עליו 300₪/חודש.

**סטטוס:** שלבים 1–3 + שלב 2 ב־Make + שלב 4א (Brand Kit) + **שלב 4ב (Preview & Edit) הושלמו.** הבא: Error handlers ב־Make, Premium (Veo/Kling).

**הבעיה היום:** המערכת מייצרת תוכן "גנרי". בעל עסק רוצה **את ההמבורגר שלו**, מתווך רוצה **את הדירה ברחוב הרצל 5** – לא דימיון AI.

**הפתרון:** להיות **"העורך האוטומטי הישראלי"** – לקחת את התמונות/הווידאו של הלקוח ולהפוך אותם לרילס עם מוזיקה וכתוביות בעברית. המנוע (Remotion + Make + Next.js) נשאר; הדלק משתנה מ־"ג'נרוט מאפס" ל־**User Content + AI Smart Editing**.

## תוכנית ביצוע (0→100)

### שלב 1: חומרי גלם (מה משנים באתר)

**מטרה:** במקום רק טקסט – המשתמש **מעלה 3–5 תמונות** (הדירה/המוצר).

| משימה | פרטים |
|--------|--------|
| **Frontend** | כפתור / Drag & Drop להעלאת תמונות בדף היצירה (3–5 קבצים). |
| **Storage** | קבצים נשמרים בענן (Firebase Storage או R2). מחזירים URLs. |
| **API** | `POST /api/upload-media` – מקבל קבצים, מעלה, מחזיר `{ urls: string[] }`. |
| **Generate** | Body של `/api/generate` + Webhook ל־Make כוללים `uploadedImageUrls?: string[]`. |
| **Make** | מקבל את ה־URLs – **שלב 2** ישלח אותם ל־Gemini Vision. |
| **תלות** | יצירת תמונה (Imagen) הופכת ל**אופציה** למי שאין לו תמונות. |

### שלב 2: שדרוג המוח (Make – Vision API)

**מטרה:** Gemini **מסתכל** על התמונות שהלקוח העלה וכותב שיווקית **עליהן**.

| משימה | פרטים |
|--------|--------|
| **Make** | במודול Gemini: לשלוח את ה־URLs של התמונות (במקום/בנוסף לטקסט). |
| **פרומפט** | "תסתכל על התמונות האלה (סלון יוקרתי / המבורגר). תכתוב 3 משפטי מחץ שיווקיים בעברית לטיקטוק שיופיעו על הסרטון." |
| **תוצאה** | טקסט מתאים למוצר האמיתי ("מטבח משופץ ברמה גבוהה") ולא גנרי. |

---

### שלב 3: שדרוג העורך (Remotion – Slideshow)

**מטרה:** וידאו מ**מערך תמונות** עם מעברים, לא תמונה אחת.

| משימה | פרטים |
|--------|--------|
| **קומפוזיציה** | מקבלת `images: string[]` במקום `imageUrl: string`. |
| **מעברים** | Slide / Fade / Zoom בין תמונות. |
| **Ken Burns** | זום איטי לכל תמונה (פנימה/החוצה). |
| **סאונד** | ללא שינוי – מוזיקה + SFX לפי ווייב. |
| **Worker** | מעביר `sourceImageUrls` (או `imageUrl` כ־fallback לתמונה אחת). |

| **Preview & Edit** | אחרי ש־Gemini מחזיר טקסט: מסך ביניים עם תיבת עריכה. משתמש מתקן → לוחץ "רנדר וידאו". |

## סיכום צ'קליסט

- [x] **שלב 1:** Frontend – Drag & Drop להעלאת תמונות; API upload (R2); שליחת `uploadedImageUrls` ל־Make ו־Firestore.
- [x] **שלב 2:** Make – **הושלם ובדוק.** Router שמקבל `uploadedImageUrls`, מעביר ל־Gemini Vision, מחזיר caption, overlayText, audioVibe (ו־videoStyle אופציונלי). ה־Backend ב־Make וה־Worker מדברים עם הנתונים של Gemini.
- [x] **שלב 3:** Remotion – קומפוזיציה עם מערך תמונות (`images`), slideshow עם Ken Burns לכל מקטע; Worker קורא `sourceImageUrls` ומעביר `images` ל־Remotion.
- [x] **שלב 4א:** Brand Kit – דף הגדרות `/settings` (לוגו, צבע מותג, טלפון, אתר); ב־Remotion: Watermark (לוגו בפינה) + Outro (מסך סיום עם לוגו, טלפון, אתר). Worker קורא מ־`users/{uid}/brandKit/settings`.
- [x] **שלב 4ב:** Preview & Edit – מסך ביניים: Make מחזיר `pending_review`; הלקוח רואה טקסט בדף היצירה וב־`/g/[genId]`, עורך ולחץ "רנדר וידאו" → `POST /api/generations/[genId]/confirm-render` → status "rendering" → Worker מרנדר.
- [ ] **Make – Error handlers:** Add error handler למודולים (Gemini, HTTP וכו') – שליחת `status: "error"` ל־callback כדי שהלקוח לא יחכה לנצח.
- [ ] **Premium (Veo/Kling):** ענף שלישי ב־Router לווידאו AI גנרטיבי (לא סליידשו).

## הבא – מה לעשות עכשיו

1. **Make – Error handlers**  
   ב־Make.com: Add error handler למודולים 3 (Gemini טקסט), 6 (Gemini תמונה), 8 (Cloudinary). בכל אחד – HTTP POST ל־`callbackUrl` עם `genId`, `status: "error"`, `errorMessage` מתאים. פרטים ב־`MAKE-ERROR-HANDLERS-AND-PREMIUM.md`.

2. **Premium – ענף ב־Make**  
   ב־Make: Router אחרי Parse JSON – אם `type === "premium"` → מודול וידאו (Veo/Kling) → HTTP Callback עם `resultUrl` (וידאו). אחרת → Gemini תמונה → Cloudinary → Callback. ה־callback הקיים באתר כבר מטפל ב־`status: "done"` + `resultUrl` גם ל־premium.

3. **בדיקה (אופציונלי)**  
   זרימה מלאה: צור יצירת remotion → Make מחזיר pending_review → ערוך טקסט בדף היצירה/דף פרטי → "רנדר וידאו" → Worker מרנדר → status "done" עם קישור.
